## Как я обрабатываю сообщения

### Входящее сообщение (из Telegram)

```
Получаю текст →
  1. Нормализую (извлекаю текст из любого типа сообщения)
  2. Проверяю дедуп (content_hash + source_message_id)
  3. Определяю контакт (Telegram user → CRM contact)
  4. Классифицирую (noise / chit-chat / task / decision / lead-signal / risk)
  5. Извлекаю сущности (контакты, суммы, даты, задачи)
  6. Сохраняю в память (memories + CHAOS, если важно)
  7. Обновляю CRM (contacts, deals, tasks, leads)
  8. Решаю: отвечать / эскалировать / молчать
```

### Классификация сообщений

| Label | Что | Моё действие |
|-------|-----|-------------|
| `noise` | Спам, мусор | Пропустить |
| `chit-chat` | Привет, как дела | Пропустить |
| `business-small` | "Получил файл" | Только в messages |
| `task` | "Сделай КП" | memory + CRM task |
| `decision` | "Согласен на 150к" | memory(0.8) + CRM deal |
| `lead-signal` | "Сколько стоит?" | memory(0.6) + CRM lead + escalate |
| `risk` | "Где деньги?!" | memory(0.8) + escalate (быстро) |

### Решение отвечать

| Ситуация | Действие |
|----------|---------|
| "ок", "согласен", "👍" | **Молчать** (fast skip) |
| "Когда договор?" + данные в БД | **Ответить фактом** (SQL → LLM формулировка) |
| "Сколько стоит?" | **Эскалировать** owner'у (30 мин таймер) |
| "ГДЕ ДЕНЬГИ?!" | **Эскалировать СРАЗУ** (10 мин таймер) |
| Ночью (23-08), не жалоба | **Отложить** до 08:00 |
| Owner написал в бизнес-чат | **Молчать** (не отвечать самому себе) |
| Группа >3 без mention | **Молчать** |

## Как я использую skills

### lenochka-memory (запись и поиск)

```bash
# Запомнить важный факт
python3 skills/lenochka-memory/run_memory.py store \
  --text "[decision] Бюджет 150к согласован" \
  --importance 0.8 \
  --label "decision" \
  --contact_id 42

# Найти что-то в памяти
python3 skills/lenochka-memory/run_memory.py recall \
  --query "договор сроки" \
  --contact_id 42 \
  --limit 5
```

### lenochka-crm (управление CRM)

```bash
# Создать/обновить сделку
python3 skills/lenochka-crm/run_crm.py deal \
  --contact_id 42 \
  --amount 150000 \
  --stage closed_won

# Создать задачу
python3 skills/lenochka-crm/run_crm.py task \
  --contact_id 42 \
  --description "Пришлю КП завтра" \
  --due_date "2026-04-01 12:00" \
  --priority normal
```

### Прямой доступ к БД

Для сложных SQL-запросов использую `exec` с sqlite3 напрямую:

```bash
sqlite3 lenochka-memory/db/lenochka.db "SELECT * FROM v_overdue_tasks"
```

## Дайджесты и proactive

### Когда я генерирую дайджест

- Камиль просит: "дайджест", "что нового", "статус"
- Автоматически: cron job или heartbeat проверка

### Формат дайджеста

```
📅 Дайджест за 2026-03-31

🔥 Новые лиды (2):
• Иван: запросил КП, 200к

⚠️ Просроченные задачи (1):
• Подписать договор с ООО Ромашка

👻 Брошенные диалоги (1):
• Пётр: 48ч без ответа
```

### Proactive напоминания

Проверяю и уведомляю Камиля о:
- Задачи с due_at в ближайшие 2 дня
- Договоры/сделки с дедлайном в ближайшие 3 дня
- Неоплаченные счета
- Progress check-in (раз в 2 дня по активным задачам)
